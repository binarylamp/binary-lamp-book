# Editorial Policy

## Mission

Making IT and English accessible to non-engineers.

## News Selection Process

毎日17時に実行。

対象分野：

- AI
- IT
- クラウド
- セキュリティ
- 通信
- ビジネス
- 若者文化

目的：

ニュースから学べるIT英語を発掘すること。

各ニュースについて

1. タイトル
2. 要約
3. なぜ面白いか
4. IT英語候補
5. 語源
6. ITでの意味
7. 初学者向け解説
8. 元記事URL

まずニュース候補と単語候補だけ提示する。


## Repository Editing Rules

Before making any changes:

- Always pull the latest version of the GitHub repository.
- Read both `editorial-policy.md` and `book-style.md`.
- Treat the GitHub repository as the single source of truth.

When creating a new article:

- Modify only the files required for that article.
- Do not rename or renumber existing articles.
- Do not reorganize the table of contents unless explicitly instructed.
- Preserve all existing content unless changes are requested.

Before committing:

- Perform a Quality Checklist.

## Quality Checklist

- Does the article start with a real-world situation?
- Does it explain the original meaning before the IT meaning?
- Does it teach an idea instead of only defining a word?
- Is the English natural rather than translation-like?
- Does it include Japanese Notes?
- Does it include the original news source and URL?
- Does it follow `book-style.md`?
- Would a beginner enjoy reading it?

Only commit if every item passes.

Never push to GitHub automatically.

Wait for my approval before pushing or publishing.
